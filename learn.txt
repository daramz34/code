Git is a Time Machine for your code. Instead of making 100 copies of a file, Git stays inside your folder and takes "snapshots" (called Commits) of your work. If you break your code on Tuesday, you can use Git to
 "travel back in time" to exactly how the code looked on Monday.

The Terminal and the Save cycle

1. git status (The "Check-in")
Before you do anything, you ask Git: "What's going on right now?"

It tells you which branch you are on.

It tells you if you have any new changes that haven't been saved yet.


2. git add . (The "Staging")
Imagine you are packing a box to send to a friend.

Putting items on the table to see what fits is git add.

The . means "Add everything in this folder."

3. git commit -m "message" (The "Snapshot")
This is closing the box and taping it shut.

The -m stands for message.

You must always describe what you did (e.g., "Added the about page"). This helps you "travel back in time" later because 
you’ll know exactly what each snapshot contains.

4. git push- to update the GitHub

5. git pull - to bring an update from github to local host

6. git commit -am "message"

7. git add nameoffile

8. git add nameoffile && commit -m "message" = this is git add and get commit in one command

Query PARAMETER
@app.get("/items/")
def search_items(
    q: str = None,                    # Optional
    category: str = "all",            # Optional with default
    max_price: float = 10000.0        # Optional with default
):
    return {"search": q, "cat": category, "max": max_price}

## Mistake 1: Confusing Path vs Query

| Wrong | Why Bad | Right |
|-------|---------|-------|
| `/user?id=42` | Query for required ID | `/user/42` |
| `/products/laptop` | Path for filter | `/products?q=laptop` |

**Rule:** Required identifier = Path. Optional filter = Query.

---

## Mistake 2: Required Query Parameter

| Wrong | Problem | Right |
|-------|---------|-------|
| `q: str` | Must provide or error | `q: Optional[str] = None` |

**Fix:** Add `Optional` and default `= None`.

---

## Mistake 3: Wrong List Index vs ID

| Wrong | Crashes/Wrong | Right |
|-------|-------------|-------|
| `return products[id]` | Index 2 ≠ ID 2 | Loop: `if p["id"] == id` |

**Fix:**
```python
for p in products:
    if p["id"] == product_id:
        return p
```

---

## Mistake 4: Modifying List While Iterating

| Wrong | Problem |
|-------|---------|
| `for p in results: if bad: results.remove(p)` | Skips items, bugs |

**Fix:** Create new list.
```python
results = [p for p in results if p["price"] <= max_price]
```

---

## Mistake 5: Case-Sensitive Search

| Wrong | Misses matches |
|-------|----------------|
| `q in p["name"]` | "Laptop" won't match "laptop" |

**Fix:**
```python
q.lower() in p["name"].lower()
```

---

## Mistake 6: Trailing Slash Confusion

| Define | Visit | Result |
|--------|-------|--------|
| `/items/` | `/items` | ❌ 404 |
| `/items` | `/items/` | ❌ 404 |

**Fix:** Pick one, use consistently. I recommend `/` at end.

---

## Mistake 7: Wrong Validation Import

| Wrong | Error |
|-------|-------|
| `from fastapi import path` | `path` not found |

**Fix:**
```python
from fastapi import Path, Query  # Capital P, Q
```

---

## Mistake 8: Mutable Default Arguments

| Wrong | Bug |
|-------|-----|
| `def f(results=[]):` | List shared between calls! |

**Fix:**
```python
def f(results=None):
    if results is None:
        results = []
```

## Quick Reference Card

| Mistake | Check |
|---------|-------|
| Path vs Query | Required = `{id}`, Optional = `?key=` |
| Optional query | Has `= None`? |
| Find by ID | Using loop, not index? |

| Trailing slash | Consistent in all endpoints? |
| Validation | `Path(..., gt=0)` not `path`? |
